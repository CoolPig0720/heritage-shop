# 《基于 Spring Boot 的非遗产品智能定制商城系统》

## 系统设计与实现说明文档（指导性）

---

## 一、系统总体设计

### 1.1 系统设计目标

本系统旨在构建一个融合**非遗文化展示、商品交易与智能定制功能**的综合在线商城平台。系统通过前后端分离架构，实现非遗产品的在线展示、销售以及个性化定制服务，同时强化非遗文化内容的传播与用户参与体验。

系统设计目标主要包括：

1. 实现一个结构清晰、功能完整的非遗产品商城系统
2. 支持多角色用户管理（管理员、商家、普通用户）
3. 将非遗文化内容与商品信息进行深度关联展示
4. 引入智能定制机制，提升用户个性化体验
5. 保证系统具备良好的可扩展性与可维护性

---

### 1.2 技术架构设计

#### 1.2.1 系统架构模式

系统采用**前后端分离架构**，通过 RESTful API 实现前后端通信。

* 前端：负责页面展示与用户交互
* 后端：负责业务逻辑处理、权限控制与数据管理
* 数据库：负责系统数据的持久化存储

整体架构逻辑如下：

```
用户浏览器
     ↓
Vue + Element Plus（前端）
     ↓ REST API
Spring Boot（后端）
     ↓
MySQL 数据库
```

---

#### 1.2.2 技术选型说明

| 层次   | 技术选型               | 说明              |
| ---- | ------------------ | --------------- |
| 前端   | Vue + Element Plus | 实现响应式界面与组件化开发   |
| 后端   | Spring Boot        | 实现业务逻辑与接口服务     |
| 数据库  | MySQL              | 存储用户、商品、文化内容等数据 |
| 接口规范 | RESTful API        | 前后端解耦，便于扩展      |
| 智能定制 | 阿里云大模型 API         | 用于生成定制设计图片      |

---

## 二、系统功能模块设计

系统功能按照业务逻辑划分为**四大核心模块**，各模块相互独立又协同工作，便于后续分阶段实现与维护。

---

## 三、用户基础模块设计

### 3.1 模块概述

用户基础模块是系统运行的核心基础模块，主要负责用户身份管理、权限控制以及用户基本信息维护，为商城系统和智能定制功能提供身份与权限支撑。

系统设定三类用户角色：

* 管理员
* 商家
* 普通用户

---

### 3.2 注册登录模块

#### 模块描述

该模块用于实现用户注册、登录与身份认证功能。系统通过角色区分，不同用户登录后进入不同的功能界面，并拥有不同的操作权限。

#### 功能设计要点

1. 用户注册

   * 用户名唯一性校验
   * 密码加密存储
   * 用户角色选择（普通用户 / 商家）

2. 用户登录

   * 用户名 + 密码登录
   * 验证码校验，防止恶意登录
   * 登录成功后生成登录状态凭证

3. 密码重置

   * 提供“忘记密码”功能
   * 支持通过验证后重置密码

#### 实现思路（指导）

* 后端通过 Spring Boot 实现登录接口
* 使用统一的权限校验机制控制访问
* 前端根据用户角色动态加载菜单和页面

---

### 3.3 个人中心模块

#### 模块描述

个人中心模块用于展示和维护当前登录用户的基本信息，增强用户对系统的可控性和使用体验。

#### 功能设计要点

* 展示用户头像、用户名、角色信息
* 支持修改登录密码
* 支持更新个人基本信息

#### 实现思路

* 前端通过接口获取当前用户信息
* 后端根据登录状态返回对应用户数据
* 修改操作需进行权限校验

---

### 3.4 用户管理 / 商品管理模块

#### 3.4.1 用户管理模块

##### 模块描述

该模块仅对管理员开放，用于对系统内所有用户进行统一管理。

##### 功能设计要点

* 用户信息查询
* 用户新增、修改、删除
* 用户角色管理

---

#### 3.4.2 商品管理模块

##### 模块描述

商品管理模块用于非遗产品的维护与管理，根据用户角色划分操作权限。

##### 功能设计要点

* 管理员：可管理系统内所有商品
* 商家：仅可管理自己发布的商品
* 支持商品新增、修改、删除、上下架

---

## 四、商城基础模块设计

### 4.1 模块概述

商城基础模块是系统的核心业务模块，主要负责非遗商品的展示、浏览与查询，为用户提供类似常见电商平台的购物体验。

---

### 4.2 商品列表展示模块

#### 模块描述

系统首页用于展示推荐非遗商品及其简要信息，提升用户对非遗产品的直观认知。

#### 功能设计要点

* 商品推荐展示
* 按商品名称、商家名称搜索
* 按非遗类别进行筛选浏览

---

### 4.3 商品详情模块

#### 模块描述

商品详情模块用于展示单个商品的详细信息，帮助用户全面了解产品特性。

#### 功能设计要点

* 商品价格、描述、图片展示
* 商品 360° / 三维展示
* 商品溯源码展示（用于查看产品来源信息）

#### 实现思路

* 商品三维展示可通过前端组件实现
* 溯源码与商品信息进行关联存储

---

## 五、非遗文化内容展示模块设计

### 5.1 模块概述

该模块用于系统性展示非遗文化内容，将文化传播与商品销售进行结合，突出系统的文化属性。

---

### 5.2 非遗文化内容展示模块

用于展示非遗项目相关文化信息，包括项目介绍、工艺流程和传承人信息。

#### 功能设计要点

* 非遗项目分类展示
* 非遗项目详细介绍
* 工艺流程图文展示
* 非遗传承人信息展示

---

### 5.3 非遗文化内容管理子模块

#### 模块描述

该子模块仅对管理员开放，用于对系统中的非遗文化内容进行统一维护。

#### 功能设计要点

* 非遗项目新增、修改、删除
* 工艺流程内容维护
* 传承人信息管理

---

## 六、智能定制模块设计

### 6.1 模块概述

智能定制模块是本系统的重要创新点，旨在通过智能化手段增强用户参与感，实现非遗产品的个性化定制体验。

---

### 6.2 智能定制模块

#### 模块描述

用户可在智能定制模块中描述定制需求，系统将用户输入信息转化为定制参数，并调用第三方智能模型生成设计图片供用户参考。

#### 功能设计要点

* 非遗类型选择（如剪纸、蜡染等）
* 风格特征选择（传统、现代、国潮等）
* 使用场景选择
* 支持用户自定义文字描述
* 展示生成的设计图片

#### 实现说明（不涉及具体 API）

* 后端负责封装第三方智能生成接口
* 前端展示生成结果并提供交互界面
* 智能定制接口的具体实现后续单独说明

---

### 6.3 定制子模块

#### 模块描述

用于用户与商家之间围绕定制需求进行进一步沟通与确认。

#### 功能设计要点

* 用户提交定制请求
* 商家查看并确认需求
* 定制状态管理（进行中 / 已完成等）

---

## 七、其他可扩展功能说明

系统在设计时预留扩展空间，可根据开发进度与实际情况添加以下功能：
